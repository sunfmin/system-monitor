# System Monitor

A [Claude Code](https://docs.anthropic.com/en/docs/claude-code) skill that captures your screen, system audio, and microphone in real time — then transcribes, summarizes, and displays everything in a live web dashboard.

Built for monitoring meetings, presentations, YouTube videos, or any on-screen activity. Claude periodically reads the screenshots and transcripts to generate running AI summaries of what's happening.

## How It Works

```
┌─────────────────────────────────────────────────────┐
│           stream-audio-whisper (single binary)       │
│                                                      │
│  ScreenCaptureKit ──► whisper.cpp C API (Metal GPU)  │
│                              │                       │
│                    ┌─────────┴──────────┐            │
│                    │                    │            │
│              2s partials          20s finals          │
│            (streaming text)    (complete sentences)   │
│                    │                    │            │
│                    └────────┬───────────┘            │
│                             │                        │
│                     WebSocket push                   │
│                      (port 8421)                     │
└─────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────┐
│              Web Dashboard (port 8420)                │
│                                                      │
│  ┌─────────────────────┐  ┌──────────────────────┐  │
│  │  Streaming partial   │  │   AI Summaries       │  │
│  │  text (top, sticky)  │  │   (right panel)      │  │
│  ├─────────────────────┤  │                      │  │
│  │  Sentence timeline   │  │   Generated by       │  │
│  │  (scrolling below)   │  │   Claude every 3min  │  │
│  └─────────────────────┘  └──────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

**Two-tier transcription** gives you the best of both worlds:
- **Tier 1 (every 2s):** Fast partial transcription for real-time subtitle display
- **Tier 2 (every 20s):** Full re-transcription with punctuation and sentence segmentation

## Features

- **Zero-config audio capture** — ScreenCaptureKit captures system audio directly. No BlackHole, no virtual audio devices, works with any output (speakers, headphones, Bluetooth)
- **Real-time subtitles** — 2-second latency streaming transcription via WebSocket
- **99 languages** — Whisper auto-detects the language, no manual selection needed
- **Smart screenshots** — Auto-detects meeting/video windows (Zoom, Teams, Lark, YouTube), captures every 30s, deduplicates similar frames
- **Window targeting** — Point it at a specific window by name (e.g., "youtube", "zoom")
- **AI summaries** — Claude reads screenshots + transcripts every 3 minutes and writes a running summary
- **Live dashboard** — Web UI with streaming subtitles, sentence timeline, and summary panel
- **Complete recording** — Full audio saved as WAV for later reference

## Requirements

- **macOS 12.3+** (for ScreenCaptureKit)
- **Apple Silicon** (pre-compiled arm64 binary included)
- **Screen Recording permission** (macOS will prompt on first run)

### Install dependencies

```bash
# whisper.cpp (provides libwhisper, libggml, and whisper-cli)
brew install whisper-cpp

# OpenCC for Traditional → Simplified Chinese conversion
pip3 install opencc-python-reimplemented
```

### Download whisper model

```bash
# Download the small model (~466MB) to the skill's models directory
curl -L "https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-small.bin" \
  -o ~/.claude/skills/system-monitor/models/ggml-small.bin
```

## Usage

### As a Claude Code skill

```bash
# Start monitoring (auto-detects the best window)
/system-monitor

# Monitor a specific window
/system-monitor youtube
/system-monitor zoom

# Check status
/system-monitor status

# Generate a summary now
/system-monitor summarize

# Stop monitoring
/system-monitor stop
```

### Manual (without Claude Code)

```bash
# Start a session
bash scripts/start-session.sh

# Start targeting a specific window
bash scripts/list-windows.sh                    # find the window ID
bash scripts/start-session.sh --window-id 1234

# Stop the session
bash scripts/stop-session.sh
```

Then open http://localhost:8420 for the live dashboard.

## Architecture

### Single-process design

The core of the system is `stream-audio-whisper.swift` — a single ~700-line Swift binary that combines:

1. **Audio capture** via ScreenCaptureKit (16kHz mono float32)
2. **Whisper inference** via the whisper.cpp C API (Metal GPU accelerated)
3. **WebSocket server** via Network.framework (port 8421)

This replaced an earlier multi-process pipeline (`capture-audio | python | whisper-cli`) that was fragile and had high latency.

### Key technical details

- **Model stays warm:** Loaded once at startup, stays in GPU memory (~0.1s load time)
- **Sentence carryover:** Incomplete sentences at the end of a 20s window carry over to the next round using precise whisper segment timestamps — no sentences get cut in half
- **Silence handling:** Silent chunks skip partial transcription but still accumulate audio for the 20s final, preventing timing drift
- **30s whisper cap:** Final interval (20s) + carryover (up to 10s) must not exceed whisper's 30s context limit
- **`initial_prompt` only for finals:** On 2s partial chunks, setting `initial_prompt` causes prompt text leakage. Only the 20s finals use it to guide punctuation
- **One stream at a time:** ScreenCaptureKit only allows one audio stream. Multiple instances will conflict and produce silence

### Recompiling from source

A pre-compiled arm64 binary is included. To rebuild:

```bash
bash scripts/compile.sh
```

This runs:
```bash
swiftc stream-audio-whisper.swift \
    -I whisper_module -I /opt/homebrew/opt/whisper-cpp/include -I /opt/homebrew/opt/ggml/include \
    -L /opt/homebrew/opt/whisper-cpp/lib -L /opt/homebrew/opt/ggml/lib \
    -lwhisper -lggml -lggml-base \
    -framework ScreenCaptureKit -framework AVFoundation -framework CoreMedia -framework Network \
    -O -o stream-audio-whisper
```

## WebSocket Protocol

The dashboard connects to `ws://localhost:8421` for real-time updates:

| Message type | Frequency | Content |
|-------------|-----------|---------|
| `partial` | Every 2s | Rolling window of streaming text |
| `final` | Every ~20s | Array of complete sentences with timestamps |
| `clear_partial` | After each `final` | Signals dashboard to fade out streaming text |
